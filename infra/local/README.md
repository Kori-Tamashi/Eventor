# Инфраструктура локального рабочего стенда

**Локальный рабочий стенд** — инстанс рабочего стенда, развернутый на вычислительной мощности разработчика, использующий виртуализацию или контейнеризацию для эмуляции зависимостей и сервисов, с ограниченной доступностью и упрощенной конфигурацией.

## Описание структуры папок

| Папка/Файл | Описание |
|------------|----------|
| `docker` | Docker-файлы. |
| `nginx` | Конфигурационные файлы веб-сервера. |
| `pgadmin` | Конфигурационные файлы PgAdmin. |
| `settings.conf` | Общий конфигурационный файл. |
| `Makefile` | Файл сборки локального рабочего стенда. |

## Инструкция к управлению стендом

Для использования локального рабочего стенда требуется предустановленный `docker`. Рабочей директорией считается папка `local`.

| Команда | Описание |
|---------|----------|
| `make up` | Поднять стенд |
| `make down` | Остановить и удалить контейнеры стенда |
| `make restart` | Перезапустить стенд |

## Описание стенда

| Контейнер | Описание | Порт доступа | План запуска | Особенности использования |
|-----------|----------|---------------|---------------|---------------------------|
| **database** | PostgreSQL 17 — основная база данных приложения. | 5432 | При старте инициализирует БД с учётными данными из переменных окружения. | Данные хранятся в `tmpfs` (оперативная память), поэтому при остановке контейнера все данные теряются. |
| **database-interface** | PgAdmin4 — веб-интерфейс для администрирования PostgreSQL. | 3002 | Загружает предустановленное подключение к `database` из монтируемого файла `servers.json`. Запускается после `database`. | Логин и пароль задаются через переменные окружения. Директория PgAdmin смонтирована в `tmpfs` — настройки не сохраняются между перезапусками. |
| **web-server** | Nginx — веб-сервер, обслуживающий статические файлы фронтенда. | 80 | При сборке двухступенчатый процесс: сначала компилируется фронтенд (Node.js), затем результат копируется в финальный образ Nginx. При запуске Nginx использует конфигурацию из монтируемого файла `nginx.conf`. | Конфигурация Nginx может быть изменена без пересборки образа через монтируемый файл. |
| **backend** | ASP.NET Core backend — основное серверное приложение. | 8080 | Запускает `dotnet run` с проектом `Eventor.csproj`, слушает порт `8080`. Переменная окружения `DATABASE_URL` задаёт строку подключения к PostgreSQL. Запускается после `database`. | Исходный код монтируется из `${BACKEND_PATH}/dotnet` внутрь контейнера — изменения в коде применяются без пересборки (режим разработки). |
