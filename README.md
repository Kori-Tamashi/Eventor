# 📄 Правила ведения репозитория

## 1. Ветви

### Основные ветви

* `main` – ветвь, содержащая конечные версии проекта.
* `develop` – ветвь, содержащая текущую разарабатываемую версию проекта.

### Дополнительные ветви

Дополнительные ветви оформляются в виде `type/#number`:
* `type` – вид изменения:
  * `task` – задание на разработку с доски.
  * `bugfix` – задание на исправление бага с доски.
* `#number` – номер задания с доски.

> Примеры: `task/#42`, `bugfix/#67`.
